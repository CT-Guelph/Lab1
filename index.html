Getting requirements is the very first step in the software lifecycle. This is the stage where all developers and stakeholders get together to truly narrow down what it is that end-users want. This process is not simply about listing features, but to discuss what the system needs to accomplish and how people will interact with it as well — even exploring any constraint the user might be experiencing. It is important to get this right because everything else builds off the foundation you create here. Clarifying the requirements from day-1, considerably reduces chances of misunderstanding and reshuffling later. In short, this stage is defining an orderly map of how the final product should be for both what the user needs from it as well as business goals.

As a user, I want to search for torrents easily using the built-in search engine, so that I do not have to find them through manual online searches.

As a user, I need an easy to use interface as I want to click through and quickly identify potential software features without fumbling.
